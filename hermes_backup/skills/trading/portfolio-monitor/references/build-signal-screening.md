# 全市场建仓信号筛选工作流

## 用途
当用户问"全市场筛选现在可以建仓的个股"时，按此工作流执行。

## 思路
### 第一层：SQLite数据库扫描（30秒全市场）
使用 `chanlun_screener.py` + `youzi_screener.py` 的策略函数，遍历数据库中所有股票的K线：
- 缠论4策略：底背驰 / 关键K线 / 三买 / 逆驰
- 游资3策略：强势股 / 低吸 / 反包
- 综合评分 = 缠论分 + 游资分
- 建仓条件：至少2个不同策略体系命中且综合分≥3

### 第二层：实时行情验证（精选股）
对第一层筛出的顶级标的，用腾讯HTTP API查实时行情，筛选出：
- 今日低开高走（低开>2%且已翻红）
- 日内收阳（价格高于开盘）
- 量能健康（0.5%<换手<8%）
- 在日内低位（价格在区间40%分位以下）

## 数据源
- **日线K线**：SQLite `~/.hermes/astock_data.db`，由 `astock-daily-sync` 维护
- **实时行情**：腾讯 `qt.gtimg.cn` (前缀sh/sz)
  ```
  http://qt.gtimg.cn/q=sh600000|sz300001|...
  返回parts[3]=现价 parts[5]=今开 parts[32]=涨跌幅 parts[33]=最高 parts[34]=最低 parts[38]=换手
  ```
- **日K线在线fallback**（DB无数据如688新股）：东财push2his
  ```
  https://push2his.eastmoney.com/api/qt/stock/kline/get?secid=0.{code}&fields2=...&klt=101&fqt=1&end=20500101&lmt=120
  ```
  ⚠️ 东财API偶发 `Connection aborted`，建议用新浪备用

## 信号分类

### 第一梯队（缠论+游资双确认）
至少1个缠论信号 + 至少1个游资信号同时命中：
- 🏆 荣信文化级别：三买+逆驰8/8+强+低吸+反包（5维度确认）
- 🏆 五方光电级别：底背驰+关键K线+三买+逆驰6/8+强+反包（6维度确认）

### 第二梯队（单体系多策略）
仅在缠论或游资单独体系内有多策略命中：
- 例如：底背驰+三买+逆驰（缠论3重）
- 例如：强势+低吸+反包（游资3重）

## 典型输出格式

按梯队展示，每只格式：
```
股票(代码) 价XX.XX +XX.XX%
  缠论: 底背驰 三买 逆驰6/8
  游资: 强势股 低吸
```
