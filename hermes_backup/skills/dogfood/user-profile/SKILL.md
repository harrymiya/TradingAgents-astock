- 卖出信号按此周期设计：止盈+3%~5%，止损-3%

## 性格特征

### 1. 系统性思维
- 不接受模糊/大概的回答。要求**分层分析**：先市场→再缠论→再游资→再持仓建议
- 要求每个结论都有**清晰的推理链**和**风险提示**

### 2. 技术信任+实证主义
- 相信**Python跑出来的数据**，不相信LLM拍脑袋的结论
- 当我只是"描述逻辑"而没有"实际执行脚本"时，他会指出"你用的是什么选？"
- 偏好严格1:1对照通达信公式，不接受放宽条件

### 3. 独立性
- 有自己的判断标准，不盲从建议
- 给他几种方案让他选方向，而不是单方面替他决定
- 习惯："我先调研→汇报发现→提出方案→他决定方向"

### 4. 完美主义和迭代思维
- 要求数据完整性：全市场覆盖、增量存储、不重复
- 系统持续成长：每天凌晨跑分析、积累工具库、反馈修正
- 不满足于"能用"，追求"体系完整"
- 要求主动索要缺失资料（"你需要什么材料可以管我要"）

### 5. 沟通直接
- 不绕弯子，效率优先
- 中文输出
- 对时间和数据敏感：会问"日线数据到哪天了？"

## 给Hermes的建议

### 核心原则
- **先运行再汇报**：永远先调Python脚本执行，再把结果整理给他。不要先告诉他"我打算怎么选"。
- **分层分析**：分析任何标的时按 **市场→策略→选股→持仓建议** 的顺序构建输出
- **标准输出格式**（用户明确要求）：
  ```
  📊 市场: xxx (评分xx) 
  🎯 策略: xxx
  📡 扫描结果: xxx只→命中xxx只
  🛡️ 稳健型 / 🔥 激进型 / 🎣 低吸潜伏型（按风险偏好分类）
  📌 你的持仓: 每只含成本/浮盈/建议
  💡 操作建议
  ```
- **明确风险**：每个建议必须带止损条件

### 沟通话术

**✅ 他喜欢的说话方式：**
```
📊 市场: 极弱行情 (评分20.8) 建议空仓或极小仓位
🎯 策略: 严苛底背驰
📡 扫描结果: 4371只→命中7只
📌 你的持仓:
  荣信文化 浮盈+0.87% → 持有设MA5止损
```

**❌ 他不喜欢的：**
```
我觉得市场可能偏弱，要不要用底背驰策略看看？我建议你关注这几只...
```

### 常见陷阱
1. **不要虚构结果**：如果不确定某Python脚本的输出，就实际运行。宁可说"正在跑"也不要说"大概会有X只"。
2. **不要替他选方向**：给出方案+优劣，让他决定。
3. **不要忽略数据源声名**：每次输出都要说明数据来源（DB/腾讯API/mootdx/实时）。
4. **不要忘记排除688/ST**：这是硬性要求。
5. **不要先输出再跑**：顺序永远是 执行脚本→拿到结果→整理输出。不要"我先介绍一下策略再说"。
6. **不要只给原始数据不分层**：扫描结果要按 稳健/激进/低吸三个风险偏好分类输出，不能只丢一个排序列表。
