# scan-pipeline — 三阶段选股流水线 v3.0

## 核心变更（v2→v3）

| 变更 | v2（旧） | v3（新） |
|:----|:--------|:--------|
| 第一阶段 | 三阴(腾讯API) + S3 + 三买 | **S3 + 三买v2**（三阴已独立） |
| 三阴数据源 | 腾讯API逐只HTTP（慢） | **SQLite窗口函数**（4秒） |
| 三阴归属 | 在流水线并集内 | **独立选股器**，不强制合并 |
| 第二阶段评分 | 通用6维(趋势/量能/波动/置信/动量/安全) | **按策略分组评分**（S3/三买各自数据区分特征） |
| 板块过滤 | 无 | 产业链DB + **通达信行业**板块热度加分 |
| 评分依据 | 凭感觉设定加分项 | **回测数据验证的真实区分特征** |
| 脚本独立性 | 需要LLM调工具执行 | **纯Python脚本**，cron一键调用 |

## 关键经验教训

### 第一阶段不能包含互相矛盾的策略
- **三阴（收跌）** 和 **S3/三买（上张/突破）** 是不同逻辑
- 强行合并在并集里 → 第二阶段无法统一评分（三阴天然chg为负，S3天然pos20低）
- 解决方案：**三阴独立出去做独立选股器**，流水线只做S3+三买

### 评分必须基于数据，不能凭感觉
- **错误示范**：凭感觉设定"S3振幅3-8%加分"→ 回测发现2-4%振幅胜率24.4%，8-12%只有11.3%，方向反了
- **正确做法**：先跑回测看成功的票长什么样，然后按区分特征设定评分
- **核心原则**：评分维度不能和第一阶段的筛选条件冲突

### 胜率要看正确指标
- **不要用"ret5>3%"当胜率**——这是大涨幅比率不是胜率
- **胜率应该看"ret5>0"**（正收益概率）
- S3 2025年胜率81%，2026年40.3%——受市场环境影响大

## 第一阶段详情

### S3超跌反弹（feat表，4秒出结果）

**条件**（基于回测数据验证）：
```python
subset = df[
    (df['pos_20d'] < 20) &              # 超跌低位（20日最低区间）
    (df['chg'] >= 3) & (df['chg'] < 7) & # 反弹启动
    (df['vr_5'] >= 1.2) & (df['vr_5'] < 2.5) &  # 温和放量
    (df['ma20_pct'] < -8)                # 远离MA20（确认未反转）
]
```

**回测胜率**：
| 时间段 | 样本 | 5日胜率(ret5>0) | 平均收益 |
|:-----|:---:|:--------------:|:-------:|
| 2025年 | 1,591次 | **81.0%** | +5.02% |
| 2026年 | 531次 | **40.3%** | -0.53% |

**真正的区分特征（2026年回测）**：
1. 振幅2-4%胜率24.4% > 4-6%胜率15.9% > 8-12%胜率11.3%
2. ma20偏离-8~-10%胜率18.4% > -15~-10%胜率12.7% > >-25%胜率9.1%
3. 量比1.2-1.5胜率16.8% > 1.5-2.0胜率11.0%

### 三买v2（DB版）

**条件**（基于缠论中枢理论）：
1. 前60根K线找波动<15%的中枢
2. 最近20天最高点突破中枢上沿3%以上
3. 当前价回落5-15%（回抽确认）
4. 当前价在中枢上沿之上
5. 量比>0.6
6. 站在MA20上方
7. 排除涨停后大跌的异常走势

**回测胜率**（Pipeline一致版本，2025-06~2026-06）：
| 指标 | 值 |
|:---|:--:|
| 5日正收益 | 43.2% |
| 5日>3% | 28.7% |
| 5日>10% | 11.8% |
| 5日<-10% | 10.4% |

### ⚠️ 三阴已独立
三阴已从流水线移除，作为独立选股器使用。详见`three-crows-screening` skill。
**三阴现在用SQLite窗口函数直接从daily_klines表计算**，不再需要腾讯API。

## 第二阶段评分

### S3评分（满分10）
| 特征 | 数据依据 | 分值 |
|:----|:--------|:---:|
| 振幅<5% | 2-4%胜率24.4% >> >8%胜率11.3% | 4分 |
| ma20偏离>-10% | -8~-10%胜率18.4% >> -25%胜率9.1% | 4分 |
| 量比<1.5 | 1.2-1.5胜率16.8% >> 2.0胜率9.4% | 3分 |

### 三买v2评分（满分10）
| 特征 | 分值 |
|:----|:---:|
| 回抽5-12%（黄金区间） | 4分 |
| 刚站稳MA20(0~5%) | 3分 |
| 站上5日线 | 2分 |
| 量比0.6-1.5（温和） | 1分 |

### 板块热度加分
| 数据源 | 加分 | 说明 |
|:------|:---:|:----|
| 产业链DB命中（AI算力/半导体等） | +5分 | 13大产业链 |
| 通达信行业命中（通信/芯片/光伏等） | +3分 | 逐类关键词匹配 |

### 通用加分（满分5）
| 特征 | 分值 |
|:----|:---:|
| 60日位<50（有上涨空间） | 2分 |
| ret1>0（次日预期收益为正） | 2分 |
| ret3>0（3日预期收益为正） | 1分 |

## 注意事项

### 评分设计原则
1. 每个策略独立评分维度，不与第一阶段筛选条件冲突
2. 基于回测数据验证的真实区分特征，不凭感觉
3. 多策略命中叠加加分（S3+三买同时命中+3）
4. 板块热度分与策略质量分独立叠加

### 数据源
| 表 | 用途 | 关键字段 |
|:---|:----|:--------|
| feat | S3过滤、评分 | code, chg, vr_5, pos_20d, ma20_pct, ma20, close |
| daily_klines | 三买v2运算 | date, open, high, low, close, volume |
| chain_stocks | 板块热度（产业链） | code → link_id → industry |
| stock_industries | 板块热度（通达信行业） | code, industry_name |
